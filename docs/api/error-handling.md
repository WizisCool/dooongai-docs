# 错误处理与重试

稳定接入不仅取决于请求成功率，也取决于失败时的处理策略。

## 常见状态码

- `400`：参数错误（字段缺失、类型不匹配、JSON 非法）
- `401`：鉴权失败（Key 错误、过期或请求头格式错误）
- `403`：权限不足（模型或功能不可访问）
- `404`：路径错误（常见为 Base URL 不包含 `/v1`）
- `429`：请求过快或额度限制
- `5xx`：上游或网关临时异常

## 重试策略建议

- `429` / `5xx`：可重试
- `400` / `401` / `403` / `404`：不要盲目重试，应先修正配置

指数退避参考：

- 第 1 次：等待 `500ms`
- 第 2 次：等待 `1000ms`
- 第 3 次：等待 `2000ms`
- 增加随机抖动，避免并发雪崩

## 幂等与可观测性

- 为每次请求记录 `request_id`（若客户端支持）
- 记录模型名、状态码、耗时，不记录明文 Key
- 报警维度建议包含错误率、超时率、429 比例
